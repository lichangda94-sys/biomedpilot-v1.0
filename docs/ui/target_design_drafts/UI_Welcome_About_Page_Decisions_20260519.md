# 萤火虫 / Firefly 启动页与关于页面 UI 决策记录

日期：2026-05-19
适用范围：UIShell / 启动页 / About 页面 / 后续 Figma 低保真设计 / Codex UI 开发
文件性质：UI 架构讨论记录与页面设计决策，不是代码实现任务

---

## 1. 背景

当前项目已经完成 UI 设计宪法 v2，并进入 UI Freeze / Consolidation 阶段。
根据当前 UI 信息架构审计，软件启动后的第一屏原本是 UI-01 本地测试登录页。该页面包含账号密码输入、注册/忘记密码、VIP/订阅/License 等占位信息。

经过讨论，当前阶段不应继续使用“伪登录页”作为第一屏。原因是：

1. 当前版本没有真实账号系统；
2. 注册、忘记密码、VIP、订阅、License 都只是 placeholder；
3. 用户测试阶段不应增加登录门槛；
4. 当前软件定位是本地桌面科研工具，已开放本地功能应默认免费测试；
5. 未来会接入账户、会员、订阅和 License，但不应在未实现时放在第一屏主流程中；
6. 第一屏更适合承担品牌识别、版本状态说明和进入工作台，而不是承担账号系统。

因此，决定将当前登录页替换为轻量启动页 / Welcome Page。

---

## 2. 页面结构决策：取消伪登录页，改为轻量启动页

### 2.1 决策

取消当前“账号密码登录页”作为第一屏。
软件启动后进入轻量启动页 / Welcome Page。
当前版本不要求用户登录，不显示账号密码输入框。
用户测试阶段默认免费使用已开放的本地功能。

### 2.2 启动页职责

启动页只承担以下职责：

- 品牌识别；
- 版本状态说明；
- 本地测试版说明；
- 进入本地工作台；
- 打开最近项目；
- 进入设置；
- 进入关于页面。

启动页不承担以下职责：

- 账号登录；
- 注册账号；
- 忘记密码；
- 购买会员；
- 管理订阅；
- 输入 License；
- 展示价格套餐；
- 展示正式商业账号系统。

### 2.3 账户、订阅、License 的放置位置

账户、VIP、订阅、License 不放在启动页主流程中。
这些内容统一移入：

```text
设置中心 → 账户与订阅
```

当前阶段该页面应标记为：

```text
Planned / Disabled / 后续版本开放
```

当前测试阶段说明：

```text
当前版本无需登录，已开放的本地功能可免费测试。账户、订阅、VIP 和 License 功能将在后续版本开放。
```

## 3. 品牌命名决策

### 3.1 主品牌

启动页主品牌使用：

```text
萤火虫 / Firefly
```

### 3.2 副标题

启动页副标题保留：

```text
BioMedPilot / 医研智析
```

说明：

- “萤火虫 / Firefly”作为产品主品牌；
- “BioMedPilot / 医研智析”作为现有项目名、副标题或产品说明保留；
- 当前图标继续沿用现有萤火虫图标，不修改图标。

### 3.3 命名理由

“萤火虫”比“萤火”更完整、更有画面感，能够更自然地承载“微光、陪伴、方向、安静、不刺眼”的品牌意象。
英文名使用 Firefly 即可，不需要使用拉丁词作为主品牌。
如果未来需要更学术化的子品牌或内部命名，可以另行考虑，但主品牌保持：

```text
萤火虫 / Firefly
```

## 5. 关于页面方向决策

### 5.1 页面定位

“关于”页面不做成普通版本信息页，而是做成带有人文色彩的项目说明页。

页面定位为：

```text
About + Project Manifesto + Community Invitation
```

中文可以理解为：

```text
关于萤火虫 / 项目理念 / 写给科研路上的同行 / 共同建设邀请
```

它不仅展示版本号，还要表达：

- 为什么叫萤火虫；
- 为什么做这个软件；
- 它想改变什么；
- 当前为什么免费开放；
- 未来为什么可能会有会员/订阅；
- 希望什么样的人参与测试、反馈、开发和改进。

### 5.2 页面标题建议

关于页面标题使用：

```text
关于萤火虫 / About Firefly
```

副标题使用：

```text
科研路上的一点微光
```

也可以在页面正文中使用：

```text
写给科研路上的同行
```

## 6. 关于页面内容结构

关于页面建议分为以下 6 个部分。

### 6.1 第一部分：品牌与项目身份

显示：

```text
[萤火虫图标]

萤火虫 / Firefly
BioMedPilot / 医研智析

0.1.0 internal beta
Developer Preview / 本地测试版
```

### 6.2 第二部分：为什么叫萤火虫

建议文案方向：

```text
萤火虫不刺眼，但能在黑暗中提供方向。

科研生活常常重复、琐碎、漫长。很多时间被消耗在实验记录、表格整理、数据清洗、文献筛选、图像处理和报告格式中。

萤火虫希望成为实验台旁的一点微光，帮助研究者把更多时间留给真正的思考和发现。
```

### 6.3 第三部分：为什么做这个软件

建议文案方向：

```text
这个项目来自真实科研训练中的反复体验：实验记录分散、数据整理重复、生信流程复杂、Meta 分析步骤繁琐，很多实验计算仍然依赖手工表格完成。

这些工作并不总是最有创造性的部分，但它们消耗了大量时间，也容易出错。

萤火虫希望把这些重复环节变得更清楚、更结构化、更容易复用。
```

### 6.4 第四部分：当前免费开放的态度

建议文案方向：

```text
当前版本中的许多本地工具会尽量保持免费可用，例如实验计算、基础记录、部分数据整理和测试级工作流。

这些功能来自一个朴素的想法：如果某些工具能减少科研训练中的机械劳动，它们就应该尽可能被更多学生、实验人员和年轻研究者使用。
```

### 6.5 第五部分：未来会员/订阅的说明

建议文案方向：

```text
未来如果接入云端 AI、外部模型、在线数据库服务、大规模计算、同步、团队协作或更复杂的自动化能力，这些功能会产生真实成本。

因此，部分高级功能可能会采用会员、订阅或 License 的方式维持运行。

但萤火虫的基础目标不会改变：让科研日常中那些重复、低效、容易出错的工作，变得更清楚、更轻一些。
```

### 6.6 第六部分：邀请参与

建议文案方向：

```text
萤火虫不是一个一次性完成的软件。它更像一个逐渐长出来的科研工具箱。

如果你在实验、数据分析、文献整理或报告生成中遇到重复、低效、容易出错的环节，欢迎把它告诉我们。

如果你懂开发、设计、生信、Meta 分析、实验流程，或者只是愿意认真测试一个功能，你都可以参与进来。

这个项目希望由真正使用它的人一起塑造。
```

## 9. 关于页面必须保留的边界说明

虽然 About 页面应具有人文色彩，但仍必须保留必要边界，避免误导用户。

页面底部或折叠说明中应包含：

```text
当前版本为 Developer Preview / 本地测试版。软件输出仅用于科研流程整理、数据检查、实验记录和内部测试，不构成临床建议、诊断结论、正式统计结论或发表级报告。
```

还应包含：

```text
当前版本以本地使用为主。AI 与外部引擎功能默认关闭或需用户自行配置。
```

这些说明可以放在页面下方的“小字说明”或“版本与边界”折叠区中，不抢占人文主叙事。
