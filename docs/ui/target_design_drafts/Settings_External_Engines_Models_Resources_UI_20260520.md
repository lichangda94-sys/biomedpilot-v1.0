# Settings UI 信息架构与外部能力管理决策

日期：2026-05-20
适用范围：UIShell / Settings / 外部引擎、模型与分析资源 / 后续 Codex UI 开发
文件性质：阶段性 UI 架构决策与 Codex 实现输入，不是完整 Figma 设计稿

---

## 1. 文件背景

本文件记录 Settings / 设置中心相关 UI 信息架构与外部能力管理决策。

此前已确认：

- Welcome Page 不承载账号、订阅、VIP、License；
- Dashboard 不显示图标资源状态、账号订阅提示或技术路径；
- 外部图像分析引擎不在用户第一层暴露具体名称；
- GO / KEGG、R/Python 分析包、绘图包等应在 Settings 中统一管理；
- 外部能力采用检测优先逻辑。

本文件用于后续 Codex 在 UIShell / Settings 中实现相关页面结构和状态逻辑。

---

## 2. Settings 总体结构

Settings / 设置中心采用五个子页面结构：

```text
Settings / 设置中心
├── 常规设置
├── 账户与订阅
├── 本地项目与存储
├── 外部引擎、模型与分析资源
└── 开发者诊断
```

建议 UI 形式：

```text
左侧二级导航
→ 右侧设置内容区
```

不要用大量平铺卡片替代设置导航。设置项会持续扩展，左侧导航更适合长期维护。

## 6. 外部引擎、模型与分析资源

### 6.1 页面定位

这是 Settings 中最复杂的页面。它不是单纯的 AI 设置页，而是外部能力总控页。

它统一管理软件调用的外部能力、模型、分析资源、统计包和绘图包。

### 6.2 内部结构

```text
外部引擎、模型与分析资源
├── 图像分析引擎
├── PDF 识别与 OCR 引擎
├── 本地语言模型
├── 云端 AI 服务
└── 分析资源与工具
```

页面顶部应有状态总览，下面分区展示各类能力。

## 7. 外部能力状态总览

页面顶部显示总检测状态。

示例：

```text
外部能力检测
上次检测：2026-05-20 10:32
整体状态：3 项可用，2 项未配置，1 项需要更新

[重新检测全部]
```

总览可显示：

- 上次检测时间；
- 整体状态；
- 可用数量；
- 未配置数量；
- 需要更新数量；
- 失败数量；
- 重新检测全部按钮。

总览只显示状态，不显示技术路径。

## 12. 分析资源与工具

### 12.1 页面定位

分析资源与工具不仅用于 GO / KEGG，也用于管理 Bioinformatics 和 Meta Analysis 运行所需的统计分析包、绘图包和可缓存分析资源。

### 12.2 内部结构

```text
分析资源与工具
├── 标准分析资源
│   ├── GO
│   └── KEGG
├── Bioinformatics 分析工具包
├── Meta Analysis 统计工具包
├── 绘图工具包
└── 本地缓存 / 版本 / 检测状态
```

### 12.3 第一版支持

第一版富集分析资源只支持：

```text
GO
KEGG
```

原因：

- 二者是最常见的基础富集资源；
- 可通过软件联网下载；
- 可缓存到本地后重复调用；
- 不需要复杂账号或权限申请。

### 12.4 暂不接入资源

暂不接入：

- 需要账号的资源；
- 需要权限申请的资源；
- 商业授权资源；
- 非必要复杂资源；
- 其他当前不必须的 gene set 平台。

### 12.5 分析包和绘图包

该区域还应管理：

- 生信分析工具包；
- 生信绘图工具包；
- Meta 统计工具包；
- Meta 绘图工具包；
- 必要 R 包；
- 必要 Python 包。

尤其是当前和未来 Bioinformatics / Meta Analysis 中实际调用的 R 包、统计包、绘图包，都必须在此处有对应展示。

### 12.6 每个条目统一字段

每个资源或工具包条目至少包含：

- 名称；
- 用途；
- 所属模块；
- 安装 / 下载状态；
- 版本，若可检测；
- 检测按钮；
- 安装 / 下载 / 更新入口，若适用；
- 本地路径，默认折叠；
- 错误提示和修复建议。

## 13. 分析资源与工具注册规则

后续 Codex 如果为 Bioinformatics、Meta Analysis 或其他分析模块新增以下外部依赖：

- R 包；
- Python 包；
- 统计分析工具包；
- 绘图工具包；
- 标准分析资源；
- 需要下载、缓存或版本检测的分析资源；

都必须同时在：

```text
Settings → 外部引擎、模型与分析资源 → 分析资源与工具
```

中添加对应管理项。

### 13.1 每个管理项必须包含

- 名称；
- 用途；
- 所属模块；
- 安装 / 下载状态；
- 版本；
- 检测按钮；
- 安装 / 下载 / 更新入口，若适用；
- 本地路径，默认折叠；
- 错误提示和修复建议。

### 13.2 禁止

禁止：

- 只在代码中隐式依赖外部分析包；
- 用户运行分析失败后才知道缺包；
- Settings 中没有对应状态展示；
- 将需要账号、权限或商业授权的资源伪装成可直接下载。

## 14. 外部能力检测优先规则

### 14.1 核心原则

外部引擎、模型与分析资源页面采用：

```text
检测优先
→ 判断状态
→ 用户确认后再下载 / 安装 / 更新 / 修复
```

默认第一动作不是下载或安装，而是检测。

### 14.2 自动检测时机

软件可以在以下时机自动进行轻量检测：

1. 软件启动后；
2. 用户进入 Settings → 外部引擎、模型与分析资源 页面时；
3. 用户点击“重新检测全部”时。

### 14.3 检测对象

自动检测包括：

- 图像分析引擎；
- PDF 识别与 OCR 引擎；
- 本地语言模型；
- 云端 AI 服务状态；
- GO / KEGG 资源；
- R / Python 分析包；
- 绘图包；
- Meta 统计工具包；
- 缓存资源；
- 版本状态；
- 路径是否失效；
- 上次检测失败原因。

### 14.4 自动检测不做的事情

自动检测只读取本地状态，不自动改变环境。

不自动执行：

- 下载；
- 安装；
- 更新；
- 删除；
- 上传；
- 启用云端 AI；
- 修改路径；
- 配置 API Key。

### 14.5 用户确认后才允许的操作

以下操作必须由用户主动点击确认：

- 下载资源；
- 安装工具包；
- 更新资源；
- 删除缓存；
- 修改路径；
- 启用模型；
- 配置云端服务；
- 重新安装；
- 查看和处理错误。

```text
名称

```

## 20. Codex 实现原则

后续 Codex 实现 Settings → 外部引擎、模型与分析资源 时，应遵守：

1. 不进入 Figma；
2. 不做高保真视觉；
3. 页面采用检测优先逻辑；
4. 软件启动或进入设置页时可自动轻量检测；
5. 自动检测不得下载、安装、更新、删除或上传数据；
6. 所有下载、安装、更新、删除、路径修改、启用模型、配置云服务等操作必须由用户确认；
7. 云端 AI 服务当前只显示后续开放，不显示 API Key、启用、购买等按钮；
8. GO / KEGG 是第一版标准分析资源；
9. 需要账号、权限或商业授权的资源暂不接入；
10. 新增 R/Python 包、统计包、绘图包或分析资源时，必须同步添加 Settings 管理项；
11. 每个条目必须有状态标签、检测逻辑、按钮、错误提示和默认折叠技术详情；
12. 技术细节不进入 Dashboard 和普通模块首页。
