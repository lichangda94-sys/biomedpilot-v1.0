# LabTools UI 信息架构讨论记录与重构决策

日期：2026-05-20
适用范围：UIShell / LabTools / Settings / 后续 Codex UI 开发
文件性质：阶段性 UI 架构决策与 Codex 实现输入，不是完整 Figma 设计稿

---

## 1. 文件背景

本文件记录本轮 LabTools / 实验工具模块 UI 信息架构讨论的确认结论。

本文件专门用于收束 LabTools 的模块级 UI 架构，重点解决：

1. LabTools 一级入口如何组织；
2. 通用计算器与实验专属计算的边界；
3. 试剂制备与实验材料记录的关系；
4. 实验模块如何按真实实验流程组织；
5. 图像分析能力如何嵌入具体实验，而不是作为一级入口；
6. 本地实验材料记录与未来云端协作的路线；
7. 通用计算器的动态公式求解范围。

---

## 2. LabTools 总体定位

LabTools / 实验工具不是普通工具卡片堆叠区。

它应被设计为：

```text
面向生物医学实验场景的本地实验工作台。
```

它的核心目标是：

- 减少重复实验计算；
- 管理试剂模板与配制记录；
- 辅助记录实验过程；
- 在具体实验模块中提供计算、记录、图像分析辅助和结果导出；
- 为未来本地实验材料记录和云端协作系统预留接口。

---

## 3. LabTools 一级结构

LabTools 一级结构确定为：

```text
LabTools / 实验工具
├── 通用计算器
├── 试剂制备
└── 实验模块
```

以下内容不作为 LabTools 一级入口：

- 图像分析辅助；
- 实验记录系统；
- 实验材料记录 / 库存记录；
- 图像分析引擎设置；
- 外部图像分析引擎；
- 完整协作系统。

这些能力分别作为：

```text
图像分析能力
→ 嵌入具体实验模块

图像分析引擎配置
→ Settings → 外部引擎、模型与分析资源 → 图像分析引擎

实验材料记录
→ LabTools 横向能力，嵌入试剂制备和实验模块

未来云端协作系统
→ 后续云端版本，不进入当前 LabTools 一级结构
```

---

## 4. LabTools 核心边界规则

确认边界：

```text
通用公式计算 → 通用计算器
具体实验场景计算 → 实验模块
试剂模板 / 本次配制 / 历史配制 → 试剂制备
实验事实记录 → 嵌入具体实验模块和横向记录层
图像分析 → 嵌入具体实验模块
图像分析引擎配置 → 全局 Settings
```

禁止将所有工具平铺为同级卡片。

## 5. 通用计算器

### 5.1 页面定位

通用计算器只包含跨实验场景、公式驱动、动态求解的基础实验计算。

它不是所有实验计算的集合。
具体实验相关计算仍归入对应实验模块。

### 5.2 第一版纳入范围

第一版纳入：

1. 质量-浓度-体积计算；
2. 稀释计算，C1V1 = C2V2；
3. 连续 / 梯度稀释；
4. 百分比浓度换算；
5. 酸碱 molarity / normality 计算；
6. RPM / RCF 离心换算；
7. 单位换算能力。

### 5.3 第二版可考虑

第二版可考虑：

1. DNA / RNA 基础换算；
2. 蛋白基础换算；
3. OD260 / OD280 基础换算。

### 5.4 不纳入通用计算器

以下内容不纳入通用计算器：

- 细胞计数 / 活率；
- BCA / Bradford；
- MTT / CCK-8 / AlamarBlue；
- ELISA；
- PCR / qPCR；
- Western Blot 上样计算；
- SDS-PAGE 配胶；
- Lane layout；
- 细胞接种计算；
- 转染体系计算；
- ELISA 标准曲线；
- qPCR Ct / ΔCt / ΔΔCt。

这些内容归入对应实验模块。

### 5.5 单位换算原则

单位换算不作为孤立能力，而是作为所有动态公式求解器的底层能力。

所有公式求解器中的变量字段都应包含单位选择器，计算时自动进行必要的单位换算。

仍可保留“单位换算”作为辅助入口，用于用户单独做简单单位转换。

单位换算应支持：

- 质量单位；
- 体积单位；
- 物质的量单位；
- 浓度单位；
- 百分比浓度单位；
- RPM / RCF；
- 必要的基础长度单位。

质量浓度与摩尔浓度之间换算必须要求分子量 MW。
如果缺少必要条件，例如 MW，系统应提示用户补充，而不是强行计算。

### 5.6 通用计算器 UI 原则

通用计算器应采用动态公式求解方式：

- 显示公式；
- 输入变量；
- 选择单位；
- 允许留空一个变量；
- 自动求解；
- 显示计算结果和简单步骤；
- 支持复制结果；
- 可选导出记录。

## 6. 试剂制备

### 6.1 页面定位

试剂制备用于管理试剂模板、生成本次配制单、保存配制记录，并可选联动实验材料记录。

它不是通用计算器，也不是完整库存系统。

### 6.2 确认流程

试剂制备采用以下流程：

```text
试剂制备首页
→ 我的试剂模板
→ 选择模板
→ 从模板配置试剂
→ 生成本次配制单
→ 保存配制记录
→ 可选加入实验材料记录
```

### 6.3 模板修改规则

本次配制流程中不允许直接临时修改原模板。

如果用户需要修改配方，应当：

```text
复制原模板
→ 生成新模板
→ 编辑新模板
→ 使用新模板进行配置试剂
```

### 6.4 设计原因

试剂模板是长期可复用结构。
本次配制是一次具体任务。
历史配制记录是已经发生的事实记录。

三者必须分开，避免用户在一次配制中无意修改长期模板。

### 6.5 试剂制备结构建议

```text
试剂制备
├── 我的试剂模板
├── 新建 / 编辑试剂模板
├── 从模板配置试剂
├── 本次配制单
├── 历史配制记录
└── 实验材料记录联动，可选
```

### 6.6 本次配制单操作

本次配制单可提供：

- 保存配制记录；
- 导出配制单；
- 保存并加入实验材料记录，可选；
- 返回模板。

### 6.7 材料记录联动

保存本次配制单后，用户可以选择：

```text
保存并加入实验材料记录
```

如果选择加入，则记录：

- 材料名称；
- 类型：自制试剂；
- 配制日期；
- 有效期，用户填写；
- 保存位置；
- 当前体积 / 数量；
- 单位；
- 操作者；
- 来源模板；
- 配制记录链接；
- 备注。

第一版材料记录联动可以只生成自制试剂记录，不强制做库存扣减。

## 7. 实验材料记录与库存追踪

### 7.1 总体定位

“实验材料记录”作为 LabTools 的横向能力，不作为一级入口。

第一阶段先做本地记录，不做云端同步、不做局域网共享、不做完整 LIMS。

### 7.2 记录对象

实验材料记录用于记录：

- 试剂；
- 抗体；
- 细胞株；
- 冻存管；
- 耗材；
- 标准品；
- 试剂盒；
- 后续可能扩展的引物、质粒等材料。

### 7.3 支持字段

建议字段包括：

- 名称；
- 类别；
- 品牌 / 公司；
- 货号；
- 批号；
- 规格；
- 当前数量；
- 单位；
- 存放位置；
- 开封日期；
- 有效期；
- 负责人；
- 状态；
- 备注。

### 7.4 库存事件

可记录事件：

- 新增；
- 编辑；
- 入库；
- 使用；
- 移动；
- 废弃；
- 过期；
- 配制生成；
- 复苏；
- 冻存；
- 导出。

每条事件可记录：

- 时间；
- 操作者；
- 材料名称；
- 材料类型；
- 操作类型；
- 变化前数量；
- 变化后数量；
- 单位；
- 原位置；
- 新位置；
- 关联实验记录；
- 备注。

### 7.5 第一阶段支持

第一阶段支持：

- 本地数据库；
- 新增材料；
- 编辑材料；
- 入库；
- 使用；
- 移动；
- 废弃；
- 状态记录；
- 导出 CSV / Markdown；
- 与试剂制备和实验模块轻度联动。

### 7.6 第一阶段不做

第一阶段不做：

- 云端同步；
- 多人协作；
- 局域网共享；
- 账号权限；
- 导师审阅；
- 库存预警推送；
- 扫码入库；
- 条码打印；
- 完整 LIMS；
- 电子签名；
- 合规审计。

### 7.7 后续路线

后续路线去掉局域网共享方案。

路线确定为：

```text
第一阶段：本地库存与实验材料记录
第二阶段：云端协作系统，未来开发
```

不再规划：

- 局域网共享；
- 本地电脑作为主机；
- NAS 共享库存数据库；
- 实验室内局域网同步。

未来云端协作系统可作为会员 / 订阅功能，支持：

- 用户账号；
- 课题组空间；
- 多人协作；
- 云端同步；
- 导师 / 管理员查看；
- 记录审查；
- 试剂库存共享；
- 细胞株状态共享；
- 权限管理。

## 8. 实验模块

### 8.1 实验模块分类

实验模块一级分类确定为：

```text
实验模块
├── 细胞实验
├── 蛋白实验
├── 核酸实验
├── 免疫与吸光度实验
└── 免疫组化
```

### 8.2 组织原则

实验模块应按真实实验目的和实验流程组织，而不是按计算方式或仪器读数组织。

具体实验相关的计算、记录、数据处理、图像分析辅助，必须放在对应实验模块中.

---

## 9. 图像分析能力归属

### 9.1 不作为一级入口

图像分析能力不作为 LabTools 一级入口。

### 9.2 嵌入具体实验模块

图像分析能力应嵌入具体实验模块中，例如：

- Western Blot → 条带图像分析辅助；
- 细胞划痕实验 → 迁移面积分析辅助；
- Transwell → 图像计数 / 区域统计辅助；
- 荧光 / 染色实验 → 荧光强度分析辅助；
- IHC → 染色强度 / 阳性区域分析辅助。

### 9.3 引擎配置位置

底层图像分析引擎的检测、配置、下载、路径管理、版本管理，不放在 LabTools 一级页面中。

统一放入：

```text
Settings → 外部引擎、模型与分析资源 → 图像分析引擎
```

LabTools 只在具体实验步骤中调用该引擎能力。

## 10. 细胞实验

### 10.1 确认结构

细胞实验模块采用：

```text
细胞实验
├── 细胞档案
├── 细胞操作记录
└── 细胞功能实验
```

### 10.2 不设置独立图像结果入口

不设置：

```text
细胞图像与结果分析
```

作为二级入口。

原因：

图像分析和结果导出不是独立实验目的，而是具体细胞功能实验中的步骤。

### 10.3 细胞档案

用于维护细胞基础信息，包括：

- 细胞名称；
- 细胞类型；
- 来源；
- 培养基；
- 血清比例；
- 抗生素；
- 培养条件；
- 推荐传代比例；
- 推荐传代范围；
- 污染检测记录；
- 冻存批次关联；
- 备注。

### 10.4 细胞操作记录

用于记录细胞维护和处理过程，包括：

- 复苏记录；
- 冻存记录；
- 传代记录；
- 接种 / 铺板记录；
- 给药 / 处理记录；
- 转染记录；
- 从上次记录创建。

这部分的本质是记录今天对细胞做了什么。

接种 / 铺板记录可以包含计算辅助，但仍归属于细胞操作记录，不属于通用计算器。

### 10.5 细胞功能实验

建议结构：

```text
细胞功能实验
├── 细胞增殖 / 活性实验
├── 迁移 / 侵袭实验
├── 荧光 / 染色实验
└── 其他细胞功能实验
```

细胞增殖 / 活性实验包括：

- MTT；
- CCK-8；
- AlamarBlue / resazurin；
- 克隆形成。

迁移 / 侵袭实验包括：

- 划痕实验；
- Transwell。

荧光 / 染色实验包括：

- AO / PI；
- 免疫荧光；
- 其他染色 / 荧光实验。

### 10.6 MTT / CCK-8 / AlamarBlue 归属

MTT / CCK-8 / AlamarBlue 归入：

```text
细胞实验 → 细胞功能实验 → 细胞增殖 / 活性实验
```

原因：

它们虽然使用 OD 或 fluorescence 读数，但实验目的属于细胞增殖 / 活性评价。

底层 OD 数据处理能力可以复用，但用户入口必须按实验目的归类。

## 11. 蛋白实验

### 11.1 确认结构

蛋白实验模块采用：

```text
蛋白实验
├── 蛋白样本与定量
└── Western Blot
```

### 11.2 BCA 归属

BCA 定量归入：

```text
蛋白实验 → 蛋白样本与定量
```

不作为通用计算器内容，也不作为蛋白实验下的独立一级工具。

### 11.3 Western Blot 内部流程

Western Blot 应作为蛋白实验下的完整实验流程模块，而不是单独的上样计算器。

推荐结构：

```text
Western Blot
├── 样本与蛋白定量
├── 上样体系计算
├── SDS-PAGE 配胶
├── Lane layout
├── 电泳 / 转膜记录
├── 封闭 / 抗体孵育记录
├── 曝光 / 图像记录
├── 条带图像分析辅助
└── 结果导出 / 实验记录
```

### 11.4 SDS-PAGE 配胶归属

SDS-PAGE 配胶应归入：

```text
实验模块 → 蛋白实验 → Western Blot
```

不作为：

- 通用计算器内容；
- 蛋白实验下的独立一级工具；
- 与 Western Blot 平级的独立流程。

原因：

SDS-PAGE 配胶通常发生在 Western Blot 上样体系计算之后，并且会影响胶板规格、孔数、上样布局和后续 lane layout。

### 11.5 Western Blot 曝光 / 图像记录

“曝光 / 图像记录”只记录图像来源和图像文件状态，不做图像分析设置。

应包含：

- 成像设备；
- 曝光方式；
- 曝光时间；
- 图像文件路径；
- 原始图片；
- 是否进行了图像编辑；
- 编辑说明；
- 备注。

“是否进行了图像编辑”用于记录用户是否对图像做过任何处理，例如：

- 反色；
- 裁剪；
- 亮度 / 对比度调整；
- 格式转换；
- 其他预处理。

该页面不记录“背景扣除方式”。

### 11.6 条带图像分析辅助

背景扣除属于图像分析步骤，应放入：

```text
Western Blot → 条带图像分析辅助
```

条带图像分析辅助包含：

- 选择图像；
- 选择分析区域；
- 固定矩形 ROI；
- 背景扣除方式；
- 条带灰度值；
- loading control / 总蛋白归一化；
- 导出分析结果。

## 12. 核酸实验

### 12.1 确认结构

核酸实验模块采用：

```text
核酸实验
├── 样本与核酸提取记录
├── PCR
└── qPCR
```

### 12.2 PCR 与 qPCR

PCR 和 qPCR 分开作为二级入口。

### 12.3 qPCR 前置流程

反转录 / cDNA 放在 qPCR 前置流程中。

qPCR 包含：

- RNA / cDNA 前置记录；
- 引物选择 / 引物记录；
- qPCR 体系计算；
- qPCR Plate Layout；
- Ct 数据导入；
- ΔCt / ΔΔCt 结果处理；
- 结果导出。

### 12.4 PCR

PCR 包含：

- 引物选择 / 引物记录；
- PCR 体系计算；
- PCR 程序记录；
- 产物记录；
- 结果导出。

### 12.5 引物记录归属

引物记录不归入“样本与核酸提取记录”。

引物应作为 PCR / qPCR 内部共用的引物记录与选择能力。

原因：

- 引物不是样本或核酸提取记录的一部分；
- 引物会被 PCR 和 qPCR 多次复用；
- 引物可能来自商业购买，也可能来自用户自主设计；
- 在具体 PCR / qPCR 实验中，用户需要从已有引物记录中选择，或临时新增引物记录。

建议：

- 在 PCR 和 qPCR 页面中都提供“选择已有引物 / 新增引物记录”入口；
- 底层共用同一个 primer registry；
- 引物记录可保存来源、靶基因、序列、Tm、产物长度、浓度、储存位置和备注。

### 12.6 qPCR Plate Layout

qPCR Plate Layout 属于 qPCR 流程内部。

它应以 96 孔板布局方式展示，类似 Western Blot lane layout 的实验排布逻辑，但以二维 plate 格式呈现。

可记录：

- 行：A–H；
- 列：1–12；
- 样本；
- 靶基因；
- technical replicate；

## 13. 免疫与吸光度实验

### 13.1 确认结构

免疫与吸光度实验结构为：

```text
免疫与吸光度实验
├── ELISA
├── 标准曲线与浓度计算
└── 结果记录与导出
```

### 13.2 删除 OD 数据整理入口

不再单独保留“OD 数据整理”作为用户可见二级入口。

原因：

- OD / 吸光度是读数形式，不是实验目的；
- MTT / CCK-8 / AlamarBlue 已归入细胞实验；
- ELISA 的 OD 处理归入 ELISA 工作流；
- BCA 的 OD 处理归入蛋白实验；
- 单独的“OD 数据整理”入口容易变成没有明确实验语义的工具堆。

### 13.3 OD / Plate Reader 数据处理

OD / plate reader 数据处理能力作为底层通用组件存在，可被具体实验流程调用，包括：

- ELISA；
- BCA；
- MTT / CCK-8；
- 其他具体实验。

但它不作为独立用户入口。

### 13.4 ELISA 工作流

ELISA 工作流包含：

- 试剂盒 / 标准品信息；
- 样本信息；
- Plate layout；
- OD 数据导入；
- Blank 扣除；
- 标准曲线拟合；
- 样本浓度反推；
- 稀释倍数校正；
- 结果表；
- 图表；
- 结果导出。

### 13.5 标准曲线与浓度计算

第一版可支持：

- 线性拟合；
- blank 扣除；
- 重复孔均值；
- CV；
- 样本浓度反推。

4PL / 5PL 可作为后续开放能力。

## 14. 免疫组化 / IHC

### 14.1 确认结构

免疫组化 / IHC 模块结构为：

```text
免疫组化 / IHC
├── 样本与切片信息
├── 染色流程记录
├── 抗体与试剂信息
├── 图像记录
├── 图像分析辅助
└── 结果记录与导出
```

### 14.2 页面定位

IHC 模块用于：

- IHC 实验记录；
- 图像记录；
- 染色结果分析辅助；
- 结果记录与导出。

它不是病理诊断系统，也不是自动判读系统。

### 14.3 样本与切片信息

记录：

- 样本编号；
- 样本来源；
- 组织类型；
- 物种；
- 疾病 / 模型信息；
- 石蜡切片 / 冰冻切片；
- 切片厚度；
- 切片日期；
- 样本批次；
- 备注。

### 14.4 染色流程记录

按 IHC 实验流程记录：

- 脱蜡 / 水化；
- 抗原修复；
- 内源性过氧化物酶封闭；
- 封闭；
- 一抗孵育；
- 二抗孵育；
- 显色；
- 复染；
- 脱水 / 透明；
- 封片。

每一步记录：

- 试剂；
- 时间；
- 温度；
- 浓度 / 稀释比例；
- 设备或条件；
- 备注；
- 异常情况。

### 14.5 抗体与试剂信息

记录：

- 目标蛋白 / marker；
- 一抗名称；
- 一抗货号；
- 一抗厂家；
- 一抗批号；
- 一抗稀释比例；
- 一抗孵育条件；
- 二抗名称；
- 二抗厂家；
- 二抗稀释比例；
- 显色系统；
- DAB 时间；
- 复染试剂；
- 封片剂。

### 14.6 图像记录

图像记录只记录图像事实，不做分析。

包含：

- 显微镜 / 扫描仪；
- 放大倍数；
- 拍摄区域；
- 图像文件路径；
- 原始图像；
- 是否进行了图像编辑；
- 编辑说明；
- 备注。

不放：

- 阳性面积阈值；
- 背景扣除；
- H-score 计算方式；
- 染色强度分级。

### 14.7 图像分析辅助

图像分析辅助用于：

- 图像选择；
- 分析区域选择；
- 阳性区域面积分析；
- 染色强度分析；
- 阈值设置；
- H-score / IHC score，后续；
- 分析结果表生成。

图像分析辅助中不单独显示“人工复核提示”。

原因：

该步骤本身就是人工参与式分析，用户需要主动选择图像、勾画或选择分析区域、确认阈值和分析范围。

### 14.8 结果记录与导出

“当前结果需要用户确认”的说明应放在：

```text
结果记录与导出
```

提示：

```text
当前结果仅用于实验记录和图像分析辅助，不构成病理诊断或临床结论。
```
