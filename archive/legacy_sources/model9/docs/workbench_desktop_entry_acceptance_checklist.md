# Workbench Desktop Entry Acceptance Checklist

This checklist covers the internal testing desktop entry baseline for the unified Workbench shell.

## Launch And Home

- [ ] App can launch.
- [ ] Home / Workbench is understandable.
- [ ] Product name is visible: `BioMedPilot / 医研智析`.
- [ ] Meta Analysis workspace entry is visible.
- [ ] Bioinformatics workspace entry is visible.
- [ ] Capability notice says this is an internal testing build, not a formal release.

## Project Actions

- [ ] `Create Project` is visible.
- [ ] `Open Project` is visible.
- [ ] `Load Demo Project` is visible.
- [ ] Create project works.
- [ ] Open project API works.
- [ ] Demo project loads.
- [ ] Current project name/path/workspace type is visible after project load.

## Workspace Navigation

- [ ] Meta workspace can be opened.
- [ ] Bioinformatics workspace placeholder is visible.
- [ ] Meta workflow navigation is visible:
  - Project Overview
  - PICO / Search
  - Literature Import
  - Screening
  - Extraction
  - Profile Readiness
  - Analysis
  - Reporting
- [ ] Placeholder pages are clearly marked as demo, planned, read-only, limited, or not implemented.
- [ ] Placeholder pages state that they do not run statistics.

## Meta Workspace Demo Areas

- [ ] Readiness panel is visible.
- [ ] Readiness disclaimer is visible.
- [ ] Row editor is visible.
- [ ] CSV save/load buttons are visible.
- [ ] UI clearly says no statistics are run.
- [ ] UI clearly says no reports are generated by row CSV actions.
- [ ] Invalid rows cannot be saved.
- [ ] Dirty rows must be saved before loading another file.

## Safety Boundaries

- [ ] No `AnalysisService` call is triggered by Workbench entry navigation.
- [ ] No runner is triggered by Workbench entry navigation.
- [ ] No NMA, HSROC, metaprop, or GLMM is implemented.
- [ ] No AI/PDF extraction is implemented.
- [ ] No PDF/Word export is implemented.
- [ ] No complex dashboard drill-down is implemented.
- [ ] No 10-profile full editing UI is implemented.

## Test Baseline

- [ ] No crash in headless tests.
- [ ] Focused Workbench tests pass.
- [ ] Full pytest passes.
- [ ] `git diff --check` passes.

Validation commands:

```bash
python3 -m compileall -q app core reporting tests
./.venv/bin/python -m pytest -q tests/test_project_navigation_model.py tests/test_workbench_shell.py
./.venv/bin/python -m pytest -q tests/test_workbench_desktop_entry.py
./.venv/bin/python -m pytest -q
git diff --check
```
