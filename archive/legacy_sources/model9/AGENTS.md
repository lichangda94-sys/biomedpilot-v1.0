# AGENTS

## 开发规则

- 仅在当前分支或当前工作目录内工作，不额外创建功能分支。
- 本仓库当前阶段只维护桌面端基础骨架，技术栈默认 Python + PySide6。
- 本轮未明确要求的业务逻辑、外部服务接入、云端依赖、界面美化都不引入。
- 新增模块时优先复用 `core/` 的基础设施，不把配置、路径、日志逻辑散落到业务目录。
- 目录职责保持清晰：`app/` 负责桌面入口和窗口，`core/` 负责通用基础设施，其余目录仅放对应领域代码。

## 测试规则

- 每完成一小部分必须先自测，再继续下一步。
- 优先执行轻量自测：语法编译、单元测试、模块导入检查。
- 新增基础设施或公共模块时，同步补最小单元测试。
- 未安装 GUI 依赖时，可以跳过界面运行测试，但要保证非 GUI 层测试可运行。

## 提交规则

- 保持小步提交，一次提交只解决一类问题。
- 提交前至少运行与改动范围匹配的自测，不带红测提交。
- 提交信息建议使用祈使句，清晰描述本次骨架或基础设施变更。
- 不回退他人已存在但与当前任务无关的修改。
